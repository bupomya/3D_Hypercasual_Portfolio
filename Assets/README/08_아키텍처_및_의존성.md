# 아키텍처 & 의존성

프로젝트에서 사용된 디자인 패턴, 스크립트 간 의존 관계, 핵심 상호작용을 정리합니다.

---

## 디자인 패턴

### 1. 싱글턴 패턴

전역적으로 하나만 존재하며, 어디서든 접근 가능한 매니저 클래스입니다.

| 클래스 | 역할 | 접근 방법 |
|---|---|---|
| `MiningManager` | 레벨 진행 관리 | `MiningManager.Instance` |
| `CurrencyManager` | 화폐 관리 | `CurrencyManager.Instance` |
| `WarningUI` | 경고 메시지 표시 | `WarningUI.Instance` |
| `ClearUI` | 레벨 클리어 메시지 표시 | `ClearUI.Instance` |
| `IdleHintUI` | Idle 힌트 표시 | `IdleHintUI.Instance` |
| `FeedbackManager` | 사운드/이펙트 재생 | `FeedbackManager.Instance` |

중복 인스턴스가 생기면 자동으로 파괴됩니다.

### 2. 옵저버 패턴 (이벤트)

이벤트를 사용하여 클래스 간 **직접 참조 없이** 통신합니다.

```csharp
// 이벤트 발행 (MiningManager)
event Action<int> OnLevelChanged;

// 구독 (BaseMiner)
MiningManager.OnLevelChanged += ApplyLevelData;
```

| 이벤트 | 발행자 | 구독자 |
|---|---|---|
| `OnLevelChanged` | MiningManager | BaseMiner, MiningZone |
| `OnCurrencyChanged` | CurrencyManager | UI (Text 컴포넌트) |
| `OnNPCRegistered` | ArriveZone | GuideArrowController |

**장점:** 새 기능을 추가할 때 기존 코드를 수정하지 않고 이벤트만 구독하면 됩니다.

### 3. 상속을 통한 추상화

공통 로직은 부모 클래스에, 차이점은 자식 클래스에서 오버라이드합니다.

```
BaseMovement (이동 공통)     → PlayerMovement (조이스틱), NPCMovement (자동)
BaseMiner    (채굴 공통)     → PlayerMiner (인벤토리), NPCMiner (자동 채굴)
TransferZone (전송 공통)     → DropZone, PickupZone, SellZone
PurchaseZone (구매 공통)     → ArriveZoneExpansionZone, ExchangeNPCZone, MiningNPCZone
```

### 4. 코루틴 기반 상태 관리

Unity의 코루틴(`IEnumerator` + `yield return`)으로 시간 기반 로직을 처리합니다.

| 코루틴 사용처 | 설명 |
|---|---|
| `Mineral` | 파괴 후 리스폰 대기 |
| `TransferZone` | 아이템 전송 간격 |
| `ExchangeManager` | NPC 교환 전체 라이프사이클 |
| `PlayerMiner` | 자동 채굴 간격 |

## 스크립트 의존성 맵

```
MiningManager (싱글턴, 레벨 관리)
│
├── MiningLevelData (ScriptableObject, 레벨 설정)
├── CurrencyManager (싱글턴, 화폐)
│
├── BaseMiner
│   ├── PlayerMiner
│   │   ├── PlayerMovement ← VirtualJoystick, IdleHintUI
│   │   ├── ItemStacker × 3 (mineral, mineralRed, money)
│   │   └── WarningUI (싱글턴)
│   └── NPCMiner
│       └── NPCMovement
│
├── MiningZone
│   └── Mineral
│
├── TransferZone (추상)
│   ├── DropZone    → GridStacker (IsFull 체크 + WarningUI)
│   ├── PickupZone  ← GridStacker
│   ├── SellZone
│   └── PurchaseZone (추상, 화폐 지불 → 일회성 액션)
│       ├── ArriveZoneExpansionZone → ArriveZone 확장
│       ├── ExchangeNPCZone → ExchangeNPCWorker 스폰
│       └── MiningNPCZone → NPCMiner 스폰
│
├── GuideArrowController
│   ├── PlayerMiner (스태커 상태 감시)
│   ├── ArriveZone (OnNPCRegistered 이벤트)
│   ├── TopDownCamera (카메라 연출)
│   └── CanvasGroup (조이스틱, 카메라 연출 시 숨김)
│
├── IdleHintUI (싱글턴)
│   ├── CanvasGroup (조이스틱 페이드)
│   └── Image (손가락 아이콘, DOTween 드래그 애니메이션)
│
├── FeedbackManager (싱글턴)
│   └── FeedbackData (ScriptableObject, 사운드+이펙트)
│
├── ConveyorLine
│   ├── DropZone (입력)
│   └── GridStacker (출력)
│
├── ExchangeManager
│   ├── GridStacker (입력/출력)
│   └── MoneyItem
│
├── LevelUpZone
│   └── MiningManager.TryUpgrade()
│
└── ItemTweenHelper (정적 유틸, DOTween 사용)
```

## 핵심 상호작용 5가지

| # | 상호작용 | 코드 흐름 |
|---|---|---|
| 1 | **채굴** | `PlayerMiner.DoMine()` → `Mineral.Mine()` → `ItemStacker.AddItem()` |
| 2 | **아이템 전송** | `DropZone.TransferOne()` → `ItemTweenHelper.MoveArc()` → `GridStacker.AddItem()` |
| 3 | **레벨업** | `LevelUpZone` → `MiningManager.TryUpgrade()` → `OnLevelChanged` → 모든 Miner/Zone 갱신 |
| 4 | **돈 획득** | `PickupZone` → `MoneyItem` 감지 → `CurrencyManager.Add()` → UI 갱신 |
| 5 | **NPC 교환** | `ExchangeManager` 코루틴 → 빨간광물 소비 → 돈 아이템 생성 |
| 6 | **가이드 화살표** | `GuideArrowController` → 단계별 타겟 전환 → 화살표 UI 회전 |
| 7 | **피드백 재생** | `FeedbackManager.Play()` → 사운드 + 이펙트 동시 재생 (오브젝트 풀링) |

## 새로운 코드 작업 시 참고사항

### 새 레벨 추가
1. `ScriptableObject/MiningLevel/`에 MiningLevelData 에셋을 만듭니다.
2. Inspector에서 수치(데미지, 보유량, Worker 프리팹 등)를 설정합니다.
3. `MiningManager`의 `levels` 배열에 추가합니다.

### 새 아이템 타입 추가
1. `StackerType` enum에 항목을 추가합니다.
2. `PlayerMiner.GetStacker()`에 새 분기를 추가합니다.
3. 새 `ItemStacker`를 플레이어에 추가합니다.

### 새 전송 존 추가
1. 반복 전송: `TransferZone`을 상속 → `CanTransfer()`, `TransferOne()` 구현
2. 일회성 구매: `PurchaseZone`을 상속 → `CanPurchase()`, `OnPurchaseComplete()` 구현
3. 씬에 BoxCollider(IsTrigger=true)와 함께 배치합니다.

### 사운드/이펙트 추가
1. `Create → Game → Feedback Data`로 FeedbackData 에셋 생성
2. AudioClip, Effect Prefab 할당
3. 스크립트에서 `FeedbackManager.Instance.Play(data, position)` 호출

### 외부 라이브러리
- **DOTween** — 거의 모든 아이템 애니메이션에 사용됩니다. 포물선 이동, 축소, 페이드 등.
- **VirtualJoystick** — 모바일 터치 입력을 처리합니다.
