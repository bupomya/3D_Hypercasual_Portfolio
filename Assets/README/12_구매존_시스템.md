# 구매존 시스템 (PurchaseZone)

> TransferZone을 상속받아 "화폐를 지불하고 일회성 액션을 실행하는" 공통 패턴을 추상화한 시스템입니다.
> 기존에 MonoBehaviour를 직접 상속받아 중복 코드가 있던 ArriveZoneExpansionZone, ExchangeNPCZone, MiningNPCZone을 리팩토링했습니다.

---

## 상속 구조

```
TransferZone (abstract)
│   ├─ delay, interval, tweenDuration, tweenHeight
│   ├─ OnTriggerEnter / Stay / Exit (타이머 + PlayerMiner 감지)
│   └─ abstract: CanTransfer(), TransferOne()
│
├── DropZone          ← 반복 전송 (아이템을 하나씩 GridStacker로)
├── PickupZone        ← 반복 전송 (GridStacker에서 하나씩 가져오기)
├── SellZone          ← 반복 전송 (한 스태커에서 다른 스태커로)
│
└── PurchaseZone (abstract)    ← 🆕 화폐 지불 후 일회성 액션
    ├─ cost (화폐 비용)
    ├─ CanTransfer() → 화폐 충분한지 체크
    ├─ TransferOne() → 돈 아이템 하나씩 제거, 완료 시 OnPurchaseComplete() 호출
    └─ abstract: CanPurchase(), OnPurchaseComplete()
    │
    ├── ArriveZoneExpansionZone   ← ArriveZone 확장
    ├── ExchangeNPCZone           ← 교환 NPC 스폰
    └── MiningNPCZone             ← 채굴 NPC 스폰
```

---

## PurchaseZone (추상 클래스)

> `Scripts/PurchaseZone.cs`

TransferZone의 반복 전송 메커니즘을 활용하여 **화폐 아이템을 하나씩 제거하는 것을 "전송"으로 모델링**합니다.
돈이 하나씩 빠져나가는 연출이 자연스럽게 적용됩니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `cost` | `int` | 구매에 필요한 화폐량 (기본 5) |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `CanTransfer()` | 이미 구매했는지, 돈이 충분한지 체크 |
| `TransferOne()` | 돈 아이템 1개 제거, 모두 제거되면 `OnPurchaseComplete()` 호출 |
| `GetMoneyValue()` | MoneyItem 프리팹의 value 값으로 필요 아이템 개수 계산 |
| `CanPurchase()` | **(abstract)** 서브클래스에서 구매 가능 조건 정의 |
| `OnPurchaseComplete()` | **(abstract)** 서브클래스에서 구매 완료 시 실행할 액션 정의 |

### 동작 흐름

```
플레이어가 존에 진입
    │
    ▼
TransferZone의 delay 타이머 작동
    │
    ▼
CanTransfer() 호출
    ├─ 이미 구매함 (purchased == true) → 중단
    ├─ CanPurchase() == false → 중단
    ├─ 돈 부족 → 중단
    └─ 조건 충족 → itemsRemaining 계산, true 반환
    │
    ▼
TransferRoutine 시작 (interval 간격으로 반복)
    │
    ▼
TransferOne() → 돈 아이템 1개 제거
    │
    ├─ itemsRemaining > 0 → 계속 반복
    └─ itemsRemaining == 0 → CurrencyManager.Spend() + OnPurchaseComplete()
```

## ArriveZoneExpansionZone

> `Scripts/ArriveZoneExpansionZone.cs`

ArriveZone의 수용 인원(maxCapacity)을 확장하고, 벽 연출 애니메이션을 실행합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `arriveZone` | `ArriveZone` | 확장 대상 ArriveZone |
| `expandAmount` | `int` | 확장할 수용량 (기본 1) |
| `destroyWalls` | `GameObject[]` | 축소 후 파괴할 벽 오브젝트 |
| `expansionWalls` | `GameObject[]` | 확장 시 나타날 벽 오브젝트 |
| `wallAnimDuration` | `float` | 벽 애니메이션 시간 (기본 0.4초) |
| `clearMessage` | `string` | 확장 완료 시 표시할 메시지 |

### 동작 흐름

```
비용 지불 완료 (OnPurchaseComplete)
    │
    ▼
arriveZone.maxCapacity += expandAmount
    │
    ▼
PlayExpansion() 실행
    ├─ destroyWalls → 축소(InBack) → 파괴
    ├─ expansionWalls → 0에서 원래 크기로 확대(OutBack)
    └─ 완료 후 ClearUI.Show(clearMessage)
```

## ExchangeNPCZone

> `Scripts/ExchangeNPCZone.cs`

비용을 지불하면 교환 NPC를 스폰합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `npcPrefab` | `GameObject` | 스폰할 NPC 프리팹 |
| `maxCarry` | `int` | NPC 최대 운반량 (기본 5) |
| `pickupInterval` | `float` | 픽업 간격 (기본 0.3초) |
| `dropInterval` | `float` | 드롭 간격 (기본 0.3초) |
| `pickupPoint` | `Transform` | NPC 픽업 위치 |
| `exchangePoint` | `Transform` | NPC 드롭 위치 |
| `sourceStacker` | `GridStacker` | 아이템 출발지 |
| `targetStacker` | `GridStacker` | 아이템 도착지 |

### 구현 포인트

- `CanPurchase()` → `spawnedNPC == null` (한 번만 스폰)
- `OnPurchaseComplete()` → NPC 인스턴스 생성 → `ExchangeNPCWorker.Setup()` → `StartWorking()`

## MiningNPCZone

> `Scripts/MiningNPCZone.cs`

비용을 지불하면 여러 채굴 NPC를 스폰합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `npcPrefab` | `GameObject` | 스폰할 NPC 프리팹 |
| `npcCount` | `int` | 스폰할 NPC 수 (기본 2) |
| `miningDamage` | `int` | NPC 채굴 데미지 (기본 1) |
| `miningRadius` | `float` | 채굴 탐지 반경 (기본 5) |
| `autoMiningInterval` | `float` | 자동 채굴 간격 (기본 1초) |
| `mineralDrop` | `GridStacker` | 채굴한 광물 드롭 위치 |
| `mineralItemPrefab` | `GameObject` | 광물 아이템 프리팹 |

### 구현 포인트

- `CanPurchase()` → `spawnedNPCs == null` (한 번만 스폰)
- `OnPurchaseComplete()` → BoxCollider 범위 내 랜덤 위치에 NPC들 스폰
- 각 NPC는 `NPCMiner.Setup()` → `StartAutoMining()`

## 새 구매존 추가 가이드

PurchaseZone을 상속받아 새 존을 만드는 단계:

```csharp
public class MyNewZone : PurchaseZone
{
    // 1. 필요한 필드 선언
    public GameObject somePrefab;
    private bool activated;

    // 2. 구매 가능 조건 정의
    protected override bool CanPurchase()
    {
        return !activated;
    }

    // 3. 구매 완료 시 실행할 액션 정의
    protected override void OnPurchaseComplete()
    {
        activated = true;
        // 원하는 동작 구현
    }
}
```

**Inspector 설정:**
- `cost` → 구매 비용 설정
- `delay` → 진입 후 대기 시간 (TransferZone에서 상속)
- `interval` → 돈 아이템 제거 간격 (TransferZone에서 상속)
- `gizmoColor` → 에디터 기즈모 색상
