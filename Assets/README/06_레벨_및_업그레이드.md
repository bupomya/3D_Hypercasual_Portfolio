# 레벨 관리 & 업그레이드 시스템

게임의 진행(레벨업)과 화폐를 관리하는 시스템입니다.

```
MiningManager    (싱글턴) ← 레벨 진행 총괄
MiningLevelData  (SO)     ← 레벨별 설정값
LevelUpZone              ← 업그레이드 트리거 영역
CurrencyManager  (싱글턴) ← 화폐 관리
```

---

## MiningManager.cs (싱글턴)

게임의 **레벨 진행을 총괄**하는 핵심 매니저입니다.

### 접근 방법

```csharp
// 어디서든 싱글턴으로 접근 가능
MiningManager.Instance.CurrentLevel   // 현재 레벨 (0부터 시작)
MiningManager.Instance.CurrentData    // 현재 레벨의 MiningLevelData
MiningManager.Instance.TryUpgrade()   // 업그레이드 시도
```

### 주요 프로퍼티

| 프로퍼티 | 타입 | 설명 |
|---|---|---|
| `CurrentLevel` | int | 현재 레벨 (0-based) |
| `CurrentData` | MiningLevelData | 현재 레벨의 설정 데이터 |
| `levels` | MiningLevelData[] | 모든 레벨 설정 배열 |

### 주요 메서드

| 메서드 | 반환 | 설명 |
|---|---|---|
| `CanUpgrade()` | bool | 다음 레벨이 존재하는지 확인합니다. |
| `TryUpgrade()` | bool | 업그레이드를 시도합니다. CurrencyManager에서 비용을 차감하고, 성공 시 레벨을 올립니다. |

### 이벤트

```csharp
event Action<int> OnLevelChanged
```

레벨이 바뀔 때 발동됩니다. 매개변수는 새 레벨 번호입니다.

**구독하는 클래스들:**
- `BaseMiner` → `ApplyLevelData()` 호출하여 채굴 수치 갱신
- `MiningZone` → Worker 프리팹 교체

---

## MiningLevelData.cs (ScriptableObject)

각 레벨의 설정값을 저장하는 **데이터 에셋**입니다.
Unity Inspector에서 직접 수정할 수 있습니다.

### 필드 목록

| 필드 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `levelName` | string | - | 레벨 표시 이름 |
| `workerPrefab` | GameObject | null | 해당 레벨의 Worker NPC 프리팹 (null이면 Worker 없음) |
| `miningSpeed` | float | 1 | 채굴 속도 배율 (현재 미사용) |
| `miningDamage` | int | 1 | 1회 채굴 시 데미지 |
| `maxCarryCount` | int | 5 | 플레이어 최대 보유 수량 |
| `miningRadius` | float | 2 | 채굴 감지 반경 |
| `autoMiningInterval` | float | 0 | 자동 채굴 간격 (초, 0이면 비활성) |
| `upgradeCost` | int | - | 다음 레벨로의 업그레이드 비용 |

### 에셋 위치

`Assets/ScriptableObject/MiningLevel/` 폴더에 3개 에셋이 있습니다:
- MiningLevel_1, MiningLevel_2, MiningLevel_3

### 새 레벨 추가 방법

1. Project 창에서 우클릭 → Create → ScriptableObject → MiningLevelData
2. Inspector에서 수치를 설정합니다.
3. `MiningManager`의 `levels` 배열에 추가합니다.

## LevelUpZone.cs

플레이어가 이 존에 들어서면 **자동으로 레벨업**을 시도합니다.

### 동작 원리

```
플레이어 진입 (OnTriggerEnter)
    → 현재 레벨 기록
    │
OnTriggerStay에서 매 프레임 체크:
    → 업그레이드 가능한가? (MiningManager.CanUpgrade())
    → 돈이 충분한가? (MoneyItem 개수 × value ≥ upgradeCost)
    → 레벨이 바뀌지 않았는가? (중복 업그레이드 방지)
    │
모두 true이면:
    → 필요한 만큼 MoneyItem을 moneyStacker에서 제거
    → MiningManager.TryUpgrade() 호출
```

### 돈 계산 방식

MoneyItem마다 `value`가 다를 수 있으므로, 필요한 금액에 도달할 때까지 하나씩 제거하며 합산합니다.

## CurrencyManager.cs (싱글턴)

게임 내 **화폐를 추적**하는 매니저입니다.

### 접근 방법

```csharp
CurrencyManager.Instance.Currency      // 현재 잔고
CurrencyManager.Instance.Add(10)       // 10원 추가
CurrencyManager.Instance.Spend(5)      // 5원 소비 (잔액 부족 시 false)
```

### 주요 메서드

| 메서드 | 반환 | 설명 |
|---|---|---|
| `Add(int amount)` | void | 화폐를 추가합니다. |
| `Spend(int amount)` | bool | 화폐를 차감합니다. 잔액 부족 시 `false`를 반환하고 차감하지 않습니다. |

### 이벤트

```csharp
event Action<int> OnCurrencyChanged
```

화폐가 변경될 때 발동됩니다. Text 컴포넌트를 업데이트하여 UI에 반영합니다.

### 참고

- `CurrencyManager`는 **숫자(잔고)** 를 관리합니다.
- 실제 돈 아이템의 물리적 존재는 `ItemStacker`와 `MoneyItem`이 담당합니다.
- `PickupZone`에서 MoneyItem을 주울 때 `CurrencyManager.Add()`가 호출됩니다.
