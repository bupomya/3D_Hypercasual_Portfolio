# 이동 시스템

캐릭터(플레이어, NPC)의 이동을 담당하는 스크립트들입니다.
상속 구조를 사용하여 공통 로직은 부모 클래스에, 입력 방식의 차이는 자식 클래스에서 처리합니다.

```
BaseMovement (추상 클래스)
├── PlayerMovement   ← VirtualJoystick 입력
└── NPCMovement      ← 목표 위치 자동 이동
```

---

## BaseMovement.cs (추상 클래스)

모든 캐릭터(플레이어, NPC)의 이동 로직을 담는 **부모 클래스**입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `moveSpeed` | float | 이동 속도 (기본값 5) |
| `rotationSpeed` | float | 회전 속도 (기본값 10) |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `Move(Vector3 direction)` | CharacterController를 사용해 캐릭터를 이동시킵니다. |
| `Rotate(Vector3 direction)` | 이동 방향으로 캐릭터를 Slerp로 부드럽게 회전시킵니다. |
| `GetMoveDirection()` | **추상 메서드**. 자식 클래스에서 어떤 입력을 사용할지 결정합니다. |

### 동작 원리

`Update()`에서 매 프레임 `GetMoveDirection()`을 호출하여 방향을 받고, 그 방향으로 `Move()`와 `Rotate()`를 실행합니다.
CharacterController와 Animator 컴포넌트가 같은 GameObject에 있어야 합니다.

## PlayerMovement.cs (BaseMovement 상속)

**플레이어 전용 이동**입니다. VirtualJoystick 플러그인에서 터치 입력을 받아 이동합니다.

### 동작 원리

- `GetMoveDirection()`에서 `VirtualJoystick.GetAxis()`로 조이스틱 입력값(x, z)을 받습니다.
- 채굴 중(`BaseMiner.IsMining == true`)이면 이동하지 않습니다.
- Animator의 `Walking` 파라미터를 입력 크기에 따라 업데이트합니다.

### Idle 힌트 처리

플레이어가 일정 시간 정지하면 `IdleHintUI`를 통해 이동 유도 힌트를 표시합니다.

- 이동 중 / 채굴 중 / MiningZone 안 → `idleTimer` 리셋, 힌트 숨김
- 정지 상태로 `idleThreshold` 초 경과 → `IdleHintUI.Show()` 호출
- 자세한 내용: [16_IdleHintUI_시스템.md](16_IdleHintUI_시스템.md)

### 의존성

- `BaseMiner` 컴포넌트 — 채굴 중 이동 잠금 + MiningZone 판단
- `VirtualJoystick` — 모바일 터치 입력
- `IdleHintUI` — Idle 힌트 표시/숨김

## NPCMovement.cs (BaseMovement 상속)

**NPC 자동 이동**입니다. 코드에서 목표 위치를 설정하면 자동으로 이동합니다.

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `SetTarget(Transform target)` | 이동할 목표를 설정합니다. |

### 동작 원리

- `GetMoveDirection()`에서 현재 위치와 Target 사이의 방향 벡터를 계산합니다.
- 목표에 도달하면(sqrMagnitude < 0.1) 자동으로 멈춥니다.
- Animator의 `Walking` 파라미터를 이동 여부에 따라 업데이트합니다.

### 사용처

- `ExchangeManager`에서 교환 NPC의 이동에 사용됩니다.
