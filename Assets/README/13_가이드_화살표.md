# 가이드 화살표 시스템 (GuideArrowController)

> 플레이어가 다음에 해야 할 일의 방향을 화살표 UI로 안내하는 튜토리얼 시스템입니다.
> 게임 진행 단계(Phase)에 따라 자동으로 가리키는 대상이 변경되며, ArriveZone 확장과 LevelUpZone 활성화 시 카메라 연출도 포함합니다.

---

## 가이드 흐름

```
[게임 시작]
    │
    ▼
1. GoToMiningZone ─── 화살표 → MiningZone
    │ (mineralStacker.Count >= maxCount)
    ▼
2. GoToDropZone ─── 화살표 → MineralDropZone
    │ (mineralStacker.Count == 0, 광물 모두 내려놓음)
    ▼
3. GoToRedPickupZone ─── 화살표 → MineralRedPickupZone
    │ (mineralRedStacker.Count > 0, 빨간 광물 획득)
    ▼
4. GoToExchangeZone ─── 화살표 → ExchangeZone
    │ (mineralRedStacker.Count == 0, 빨간 광물 모두 내려놓음)
    ▼
5. GoToMoneyPickup ─── 화살표 → MoneyItem 위치
    │ (moneyStacker.Count > 0, 돈 획득)
    ▼
6. Complete ─── 화살표 비활성화
    │
    ▼
카메라가 LevelUpZone으로 이동
    → LevelUpZone이 DOScale 애니메이션으로 나타남
    → 일정 시간 후 카메라 복귀
```

## GuideArrowController

| 필드 | 타입 | 설명 |

| `bobDistance` | `float` | 왕복 이동 거리 (px, 기본 15) |

| `miningZoneTarget` | `Transform` | MiningZone 위치 |
| `dropZoneTarget` | `Transform` | MineralDropZone 위치 |
| `redPickupZoneTarget` | `Transform` | MineralRedPickupZone 위치 |
| `exchangeZoneTarget` | `Transform` | ExchangeZone 위치 |
| `moneyPickupTarget` | `Transform` | MoneyItem 출현 위치 |
| `arriveZone` | `ArriveZone` | ArriveZone 참조 (확장 감지용) |
| `expansionZoneObject` | `GameObject` | ArriveZoneExpansionZone (처음에 비활성화) |
| `levelUpZoneObject` | `GameObject` | LevelUpZone (처음에 비활성화) |
| `topDownCamera` | `TopDownCamera` | 카메라 연출용 |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `CheckPhaseTransition()` | 매 프레임 스태커 상태를 체크하여 단계 전환 |

| `CameraPanRoutine()` | 범용 카메라 팬 코루틴 |

## 화살표 동작 원리

### 위치 계산

```
1. 플레이어 월드 좌표 → Camera.WorldToScreenPoint → 스크린 좌표
2. 타겟 월드 좌표 → Camera.WorldToScreenPoint → 스크린 좌표
3. 방향 벡터 = (타겟 스크린 좌표 - 플레이어 스크린 좌표).normalized
4. 화살표 위치 = 플레이어 스크린 좌표 + screenOffset + (방향 × bobOffset)
```

### 회전 계산

```
1. 방향 벡터의 각도 = Atan2(dir.y, dir.x) × Rad2Deg - 90°
   (기본 이미지가 Vector2.up을 가리키므로 -90° 보정)
2. Quaternion.Lerp로 부드럽게 회전 보간
```

### 왕복 애니메이션 (Bob)

```
DOTween.To(bobOffset, 0 → bobDistance)
    .SetEase(InOutSine)
    .SetLoops(-1, Yoyo)

→ 화살표가 가리키는 방향으로 앞뒤 왕복 이동
→ 자연스러운 "여기로 가세요" 느낌
```

## ArriveZone 확장 감지

> ArriveZone에 `OnNPCRegistered` 이벤트가 추가되어, NPC가 등록될 때만 체크합니다.
> (매 프레임 체크 대신 이벤트 기반으로 성능 최적화)

```
ArriveZone.RegisterNPC()
    │
    ▼
OnNPCRegistered 이벤트 발행
    │
    ▼
GuideArrowController.CheckExpansionZone()
    │
    ├─ CurrentCount < maxCapacity → 무시
    └─ CurrentCount >= maxCapacity → ShowExpansionZoneRoutine() 시작
```

## LevelUpZone 활성화 연출

```
MoneyItem 획득 → CompleteGuide()
    │
    ▼
화살표 비활성화
    │
    ▼
ShowLevelUpZoneRoutine() 시작
    │
    ▼
HideJoystick() ← 조이스틱 페이드아웃 + 입력 차단
    │
    ▼
카메라가 LevelUpZone 위치로 이동
    │ (cameraPanDuration 대기)
    ▼
LevelUpZone.SetActive(true)
    │ Scale(0,0,0) → 원래 크기 (OutBack 이징, 0.5초)
    │ (cameraPanDuration 대기)
    ▼
카메라가 플레이어에게 복귀
    │
    ▼
ShowJoystick() ← 조이스틱 페이드인 + 입력 복원
```

> LevelUpZone과 동일한 연출 패턴을 사용합니다.

```
    │
    ▼
ShowExpansionZoneRoutine() 시작
    │
    ▼
HideJoystick() ← 조이스틱 페이드아웃 + 입력 차단
    │
    ▼
카메라가 ExpansionZone 위치로 이동
    │ (cameraPanDuration 대기)
    ▼
ExpansionZone.SetActive(true)
    │ Scale(0,0,0) → 원래 크기 (OutBack 이징, 0.5초)
    │ (cameraPanDuration 대기)
    ▼
카메라가 플레이어에게 복귀
    │
    ▼
ShowJoystick() ← 조이스틱 페이드인 + 입력 복원
```

## Unity 에디터 설정 가이드

1. **Canvas 생성**: Screen Space - Overlay 모드
2. **화살표 Image 추가**: 위쪽(Vector2.up)을 가리키는 화살표 이미지를 Sprite로 등록
3. **빈 GameObject에 GuideArrowController 추가** 후 Inspector에서 연결:

| Inspector 항목 | 연결 대상 |
|---|---|
| Arrow Rect | 화살표 Image의 RectTransform |
| Player | PlayerMiner 컴포넌트가 있는 오브젝트 |
| Mining Zone Target | MiningZone의 Transform |
| Drop Zone Target | MineralDropZone의 Transform |
| Red Pickup Zone Target | MineralRedPickupZone의 Transform |
| Exchange Zone Target | ExchangeZone의 Transform |
| Money Pickup Target | MoneyItem이 생기는 위치의 Transform |
| Arrive Zone | ArriveZone 컴포넌트 |
| Expansion Zone Object | ArriveZoneExpansionZone GameObject (비활성화 상태) |
| Level Up Zone Object | LevelUpZone GameObject (비활성화 상태) |
| Top Down Camera | TopDownCamera 컴포넌트 |
| Joystick Canvas Group | Native Joystick (Black)의 CanvasGroup |
