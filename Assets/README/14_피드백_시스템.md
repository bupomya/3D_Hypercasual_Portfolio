# 피드백 시스템 (FeedbackManager)

> 사운드와 이펙트를 하나의 ScriptableObject로 묶어 관리하는 시스템입니다.
> 한 줄의 코드로 사운드 + 이펙트를 동시에 재생하며, 이펙트 오브젝트 풀링으로 성능을 관리합니다.

---

## 구조 개요

```
FeedbackData (ScriptableObject)
│   ├─ AudioClip clip          ← 재생할 사운드
│   ├─ float volume            ← 볼륨 (0~1)
│   ├─ float pitchMin/Max      ← 랜덤 피치 범위
│   ├─ GameObject effectPrefab ← 이펙트 프리팹
│   └─ float effectDuration    ← 이펙트 자동 회수 시간
│
FeedbackManager (Singleton)
│   ├─ Play(FeedbackData, Vector3)  ← 사운드 + 이펙트 동시 재생
│   ├─ PlaySound()                  ← 사운드만 재생
│   ├─ PlayEffect()                 ← 이펙트만 재생
│   └─ 오브젝트 풀 (Dictionary<prefab, Queue>)
│
사용 예시:
│   FeedbackManager.Instance.Play(mineFeedback, transform.position);
```

---

## FeedbackData (ScriptableObject)

> `Scripts/FeedbackData.cs`

사운드와 이펙트를 하나로 묶는 데이터 에셋입니다.
Project 창에서 `우클릭 → Create → Game → Feedback Data`로 생성합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `clip` | `AudioClip` | 재생할 사운드 클립 (null이면 사운드 미재생) |
| `volume` | `float` | 볼륨 (0~1, 기본 1) |
| `pitchMin` | `float` | 최소 피치 (기본 1.0) |
| `pitchMax` | `float` | 최대 피치 (기본 1.0) |
| `effectPrefab` | `GameObject` | 이펙트 프리팹 (null이면 이펙트 미재생) |
| `effectDuration` | `float` | 이펙트 표시 후 풀에 반환할 시간 (기본 1초) |

### 피치 랜덤 범위

> `pitchMin`과 `pitchMax`를 다르게 설정하면 재생할 때마다 약간 다른 음높이로 들립니다.
> 같은 사운드가 반복 재생될 때 단조로움을 방지하는 효과가 있습니다.
>
> 예: `pitchMin = 0.9`, `pitchMax = 1.1` → 재생마다 0.9~1.1 사이 랜덤 피치

## FeedbackManager (Singleton)

> `Scripts/FeedbackManager.cs`

사운드 재생과 이펙트 스폰을 담당하는 매니저입니다.
이펙트는 오브젝트 풀링으로 관리하여 반복적인 Instantiate/Destroy를 방지합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `initialPoolSize` | `int` | 초기 풀 크기 (기본 5) |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `Play(FeedbackData, Vector3)` | 사운드 + 이펙트를 한 번에 재생 |
| `PlaySound(FeedbackData)` | AudioSource.PlayOneShot으로 사운드 재생 |
| `PlayEffect(FeedbackData, Vector3)` | 풀에서 이펙트 꺼내 위치에 스폰 |
| `GetFromPool(GameObject)` | 풀에서 오브젝트 가져오기 (없으면 새로 생성) |
| `ReturnToPoolAfter(prefab, instance, delay)` | 일정 시간 후 이펙트를 풀에 반환 |

### Null 안전

```
Play(data, position)
  │
  ├─ data == null → 아무것도 실행하지 않음
  │
  ├─ PlaySound()
  │   └─ data.clip == null → 사운드 미재생
  │
  └─ PlayEffect()
      └─ data.effectPrefab == null → 이펙트 미재생
```

> 사운드만 있는 FeedbackData, 이펙트만 있는 FeedbackData, 둘 다 있는 FeedbackData 모두 가능합니다.

### 오브젝트 풀링 동작

```
Play() 호출
    │
    ▼
GetFromPool(prefab)
    ├─ 풀에 비활성 오브젝트 있음 → 재사용
    └─ 풀이 비어있음 → Instantiate로 새로 생성
    │
    ▼
이펙트 위치 설정 + SetActive(true)
    │
    ▼
ReturnToPoolAfter(effectDuration) 코루틴 시작
    │ (effectDuration 초 대기)
    ▼
SetActive(false) → 풀에 반환 (Enqueue)
```

## 사용 방법

### 1단계: FeedbackManager 씬에 추가

1. 빈 GameObject 생성 → `FeedbackManager` 컴포넌트 추가
2. 씬에 하나만 존재해야 함 (Singleton)

### 2단계: FeedbackData 에셋 생성

1. Project 창 우클릭 → `Create → Game → Feedback Data`
2. 이름을 알아보기 쉽게 설정 (예: `FB_MineHit`, `FB_MineDestroy`, `FB_LevelUp`)
3. 원하는 AudioClip과 Effect Prefab 할당

### 3단계: 스크립트에서 사용

```csharp
public class Mineral : MonoBehaviour
{
    // Inspector에서 FeedbackData 에셋 할당
    public FeedbackData hitFeedback;       // 채굴 타격 시
    public FeedbackData destroyFeedback;   // 광물 파괴 시

    public bool Mine(int damage)
    {
        health -= damage;

        if (health <= 0)
        {
            // 파괴 피드백 재생
            FeedbackManager.Instance.Play(destroyFeedback, transform.position);
            gameObject.SetActive(false);
            return true;
        }

        // 타격 피드백 재생
        FeedbackManager.Instance.Play(hitFeedback, transform.position);
        return false;
    }
}
```

## 활용 예시

| 상황 | FeedbackData 이름 예시 | clip | effectPrefab |
|---|---|---|---|
| 광물 타격 | `FB_MineHit` | 곡괭이 소리 | 파편 파티클 |
| 광물 파괴 | `FB_MineDestroy` | 부서지는 소리 | 폭발 파티클 |
| 아이템 픽업 | `FB_ItemPickup` | 줍는 소리 | 반짝임 파티클 |
| 레벨업 | `FB_LevelUp` | 팡파르 소리 | 축하 파티클 |
| NPC 스폰 | `FB_NPCSpawn` | 등장 소리 | 연기 파티클 |
| 돈 획득 | `FB_MoneyGet` | 코인 소리 | 코인 파티클 |

> 같은 FeedbackData를 여러 오브젝트에서 공유할 수 있습니다.
> 예를 들어 `FB_ItemPickup`을 PickupZone과 SellZone에서 모두 사용할 수 있습니다.
