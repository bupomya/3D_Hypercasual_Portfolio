# 전송 존 시스템 (Transfer Zones)

플레이어가 특정 영역에 들어서면 아이템이 자동으로 이동하는 시스템입니다.
추상 클래스 `TransferZone`을 기반으로 반복 전송 존과 일회성 구매 존이 구현되어 있습니다.

```
TransferZone (추상 클래스)
├── DropZone     ← 플레이어 인벤토리 → GridStacker (내려놓기)
├── PickupZone   ← GridStacker → 플레이어 인벤토리 (줍기)
├── SellZone     ← 플레이어 인벤토리 → 플레이어 인벤토리 (변환)
│
└── PurchaseZone (추상 클래스) ← 화폐 지불 후 일회성 액션
    ├── ArriveZoneExpansionZone   ← ArriveZone 확장
    ├── ExchangeNPCZone           ← 교환 NPC 스폰
    └── MiningNPCZone             ← 채굴 NPC 스폰
```

> PurchaseZone 계열의 상세 설명은 [12_구매존_시스템.md](12_구매존_시스템.md)를 참고하세요.

## TransferZone.cs (추상 클래스)

아이템이 한 곳에서 다른 곳으로 이동하는 **모든 존의 부모 클래스**입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `delay` | float | 진입 후 전송 시작까지 대기 시간 (기본값 1초) |
| `interval` | float | 아이템 간 전송 간격 (기본값 0.3초) |
| `tweenDuration` | float | 아이템 이동 애니메이션 시간 (기본값 0.4초) |
| `tweenHeight` | float | 포물선 이동의 최고 높이 (기본값 2) |

### 동작 원리

```
플레이어가 트리거 영역에 진입 (OnTriggerEnter)
    → currentMiner에 PlayerMiner 참조 저장
    │
OnTriggerStay에서 매 프레임 체크:
    → CanTransfer() == true이면 전송 코루틴 시작
    │
전송 코루틴:
    → delay초 대기
    → CanTransfer()가 true인 동안 반복:
        → TransferOne() 호출 (아이템 1개 이동)
        → interval초 대기
    │
플레이어가 영역을 벗어남 (OnTriggerExit)
    → currentMiner 참조 해제
```

### 추상 메서드 (자식 클래스에서 구현)

| 메서드 | 설명 |
|---|---|
| `bool CanTransfer()` | 전송 가능 여부를 판단합니다. |
| `void TransferOne()` | 아이템 1개를 실제로 이동시킵니다. |

## DropZone.cs — 플레이어 → GridStacker

플레이어 인벤토리의 아이템을 GridStacker로 **내려놓는** 존입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `targetStacker` | GridStacker | 아이템이 도착할 목적지 |
| `sourceStackerType` | StackerType | 플레이어의 어떤 인벤토리에서 꺼낼지 |
| `dropTweenDuration` | float | 드롭 애니메이션 시간 |
| `dropTweenHeight` | float | 드롭 포물선 높이 |

### 동작 흐름

```
CanTransfer() 체크:
    → source 없음 / 비어있음 → 전송 안 함
    → targetStacker.IsFull → WarningUI 경고 표시 + 전송 안 함
    → 조건 충족 → 전송 시작
    │
PlayerMiner의 ItemStacker에서 PopItem()
    → ItemTweenHelper.MoveArc()로 포물선 애니메이션
    → GridStacker.AddItem()으로 격자에 배치
```

### 특수 기능

- `tweenCount`로 현재 진행 중인 트윈 애니메이션 수를 추적합니다.
- `HasPendingTweens` 프로퍼티: ConveyorLine이 이 값을 확인하여, 아이템이 아직 날아가는 중이면 처리를 시작하지 않습니다.
- `targetStacker.IsFull`이면 WarningUI로 "보관함이 가득 찼습니다!" 경고를 표시합니다.

## PickupZone.cs — GridStacker → 플레이어

GridStacker의 아이템을 플레이어 인벤토리로 **줍는** 존입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `sourceStacker` | GridStacker | 아이템을 꺼낼 원본 |
| `targetStackerType` | StackerType | 플레이어의 어떤 인벤토리에 넣을지 |

### 동작 흐름

```
GridStacker에서 PopItem()
    → ItemTweenHelper.MoveArc()로 포물선 애니메이션
    → PlayerMiner의 ItemStacker로 이동
    → 도착 시 스택 위치에 스냅
```

### 특수 기능: 돈 아이템 감지

아이템에 `MoneyItem` 컴포넌트가 있으면:
- `CurrencyManager.Instance.Add(moneyItem.value)`를 호출하여 화폐를 추가합니다.
- 이를 통해 돈 아이템을 주우면 자동으로 잔고가 늘어납니다.

## SellZone.cs — 인벤토리 간 변환

한 종류의 플레이어 인벤토리에서 다른 종류로 **변환**하는 존입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `sourceStackerType` | StackerType | 소비할 인벤토리 타입 |
| `targetStackerType` | StackerType | 생성할 인벤토리 타입 |

### 동작 흐름

```
source ItemStacker에서 RemoveItem()
    → target ItemStacker에 AddItem()
```

별도의 이동 애니메이션 없이 즉시 변환됩니다.
예: 빨간 광물(MineralRed) → 돈(Money)

## 존 추가 가이드

### 반복 전송 존 추가 (아이템을 계속 옮기는 존)

1. `TransferZone`을 상속하는 새 클래스를 만듭니다.
2. `CanTransfer()`를 구현합니다 — 언제 전송이 가능한지 조건을 정의합니다.
3. `TransferOne()`을 구현합니다 — 아이템 1개의 이동 로직을 작성합니다.
4. 씬에 빈 GameObject를 만들고, BoxCollider(IsTrigger=true)와 새 스크립트를 추가합니다.

### 일회성 구매 존 추가 (비용을 지불하고 한 번 실행하는 존)

1. `PurchaseZone`을 상속하는 새 클래스를 만듭니다.
2. `CanPurchase()`를 구현합니다 — 구매 가능 조건을 정의합니다.
3. `OnPurchaseComplete()`를 구현합니다 — 구매 완료 시 실행할 액션을 작성합니다.
4. 씬에 빈 GameObject를 만들고, BoxCollider(IsTrigger=true)와 새 스크립트를 추가합니다.

> 상세한 구매 존 구현 가이드는 [12_구매존_시스템.md](12_구매존_시스템.md)를 참고하세요.
