# 채굴 시스템

게임의 핵심 메커니즘인 채굴을 담당하는 스크립트들입니다.
캐릭터가 광물을 감지하고, 데미지를 주어 파괴하고, 아이템을 획득하는 과정을 처리합니다.

```
BaseMiner (추상 클래스)
├── PlayerMiner   ← 플레이어 직접 채굴 + 인벤토리
└── NPCMiner      ← (미구현, Worker 프리팹이 대신 담당)

MiningZone        ← 채굴 구역 정의 + 광물 스폰 + Worker 관리
Mineral           ← 개별 광물 오브젝트
```

---

## BaseMiner.cs (추상 클래스)

채굴 가능한 모든 캐릭터의 **공통 기능**을 담당합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `humanVisuals` | GameObject[] | 캐릭터 외형 오브젝트들 |
| `miningTool` | GameObject | 채굴 시 표시되는 도구 (곡괭이 등) |
| `miningPos` | Transform | 채굴 감지 중심점 |
| `miningRadius` | float | 채굴 감지 반경 (기본값 2) |
| `mineralLayer` | LayerMask | 광물을 감지할 레이어 마스크 |

### 주요 프로퍼티

| 프로퍼티 | 타입 | 설명 |
|---|---|---|
| `IsMining` | bool | 현재 채굴 중인지 여부 |
| `IsInMiningZone` | bool | MiningZone 안에 있는지 여부 (IdleHintUI 억제용) |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `ShowHuman(bool show)` | 캐릭터 외형을 보이거나 숨깁니다. |
| `SetMining(bool mining)` | 채굴 애니메이션 시작/중지 및 도구 표시를 토글합니다. |
| `SetInMiningZone(bool value)` | MiningZone 진입/이탈 시 호출. `IsInMiningZone` 플래그를 설정합니다. |
| `FindNearestMineral()` | `Physics.OverlapSphere`로 `miningPos` 기준 반경 내 가장 가까운 광물을 찾습니다. |
| `ApplyLevelData(int level)` | MiningLevelData에서 데미지, 반경 등의 수치를 가져와 적용합니다. |
| `StartAutoMining()` / `StopAutoMining()` | 자동 채굴 시작/중지 (virtual, 자식 클래스에서 구현) |

### 핵심 원리

`MiningManager.OnLevelChanged` 이벤트를 구독합니다.
레벨이 바뀌면 자동으로 `ApplyLevelData()`가 호출되어 채굴 수치가 갱신됩니다.

에디터에서 Scene 뷰의 빨간 와이어 구체가 `miningRadius`를 시각적으로 보여줍니다.

## PlayerMiner.cs (BaseMiner 상속)

플레이어 전용 채굴기입니다. 3개의 인벤토리(ItemStacker)를 보유합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `mineralStacker` | ItemStacker | 일반 광물 인벤토리 |
| `mineralRedStacker` | ItemStacker | 빨간 광물 인벤토리 |
| `moneyStacker` | ItemStacker | 돈 인벤토리 |

### 주요 메서드

| 메서드 | 설명 |
|---|---|
| `GetStacker(StackerType type)` | 타입별로 적절한 ItemStacker를 반환합니다. |
| `OnMiningHit()` | **애니메이션 이벤트**에서 호출. 곡괭이가 내려치는 순간 실제 채굴(`DoMine()`)을 실행합니다. |
| `ApplyLevelData(int level)` | 부모의 수치 적용 + `maxCarryCount`를 스택커에 설정합니다. |

### 채굴 흐름

```
OnMiningHit() (애니메이션 이벤트)
    → DoMine()
        → FindNearestMineral()  (가장 가까운 광물 탐색)
        → mineral.Mine(damage)  (데미지 적용)
        → mineralStacker.AddItem()  (인벤토리에 추가)
```

- 인벤토리가 가득 차면 `WarningUI.Instance.Show()`로 경고를 표시합니다.
- 자동 채굴: `StartAutoMining()` 호출 시 코루틴이 `autoMiningInterval` 간격으로 `DoMine()`을 반복합니다.

## NPCMiner.cs (BaseMiner 상속)

현재 **비어 있는 구현체**입니다.
NPC의 실제 채굴은 `MiningZone`이 Worker 프리팹을 스폰하여 처리합니다.

---

## MiningZone.cs

물리적인 채굴 영역을 정의하고, 광물 스폰 및 Worker 관리를 담당합니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `mineralPrefab` | GameObject | 스폰할 광물 프리팹 |
| `mineralParent` | Transform | 스폰된 광물의 부모 트랜스폼 |
| `mineralSpacing` | float | 광물 간 간격 (기본값 3) |

### 광물 스폰

`SpawnMinerals()`에서 BoxCollider의 크기를 기준으로 **그리드 형태**로 광물을 배치합니다.
- 콜라이더 범위를 `mineralSpacing`으로 나누어 행/열 개수를 계산합니다.
- 월드 좌표로 변환하여 정확한 위치에 생성합니다.

### Worker 시스템 (핵심!)

플레이어가 구역에 진입/이탈할 때 모델을 교체하는 시스템입니다.

| 상황 | 레벨 0 | 레벨 1+ |
|---|---|---|
| **구역 진입** | 플레이어 모델 그대로 표시 | 플레이어 모델 숨김 → Worker 프리팹 스폰 |
| **구역 이탈** | 변화 없음 | Worker 파괴 → 플레이어 모델 다시 표시 |

- `MiningManager.OnLevelChanged` 이벤트를 구독하여, 레벨이 바뀌면 Worker를 교체합니다.
- 트리거(BoxCollider, IsTrigger=true)를 사용합니다.

## Mineral.cs

개별 광물 오브젝트입니다.

### 주요 필드

| 필드 | 타입 | 설명 |
|---|---|---|
| `maxHealth` | int | 내구도, 파괴에 필요한 타격 수 (기본값 3) |
| `respawnTime` | float | 파괴 후 재생성까지 대기 시간 (기본값 5초) |

### 주요 메서드

| 메서드 | 반환 | 설명 |
|---|---|---|
| `Mine(int damage)` | bool | 광물에 데미지를 주고, 체력이 0 이하가 되면 `true`를 반환합니다. |

### 라이프사이클

```
활성 상태 → Mine() 호출 → 체력 감소
    → 체력 <= 0 → SetActive(false) → respawnTime 초 대기 (코루틴)
    → SetActive(true) → 체력 복구 → 다시 활성 상태
```
